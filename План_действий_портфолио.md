# План действий: портфолио презентаций за 1 месяц

## Цель

Создать отдельный сайт-портфолио для продажи услуг по презентациям на Профи.ру и вне платформы.

Каждый сильный кейс должен иметь 3 формата:

- Figma: исходный дизайн и демонстрация визуального уровня.
- PPTX: скачиваемый рабочий файл для заказчиков.
- HTML: интерактивная веб-версия с анимацией.

Минимум, который уже можно продавать: сайт + 3 полностью готовых кейса.
Идеальный результат месяца: сайт + 5 готовых кейсов + 3 заготовки будущих кейсов.

## Позиционирование

Основная идея:

> Презентации, которые производят впечатление.

Акцент:

- бизнес-презентации;
- коммерческие предложения;
- инвестиционные питчи;
- презентации продуктов;
- анимация;
- PowerPoint;
- HTML-презентации;
- Figma-дизайн.

Стиль: премиальный, минималистичный, современный. Вдохновляться Apple можно, копировать Apple нельзя.

## Форматы результата

Для каждого полного кейса:

- 10 слайдов в Figma;
- PPTX-версия;
- HTML-версия;
- PDF-экспорт;
- 5-7 изображений ключевых слайдов;
- видео-превью 10-15 секунд;
- короткое описание кейса для сайта и Профи.ру.

## Структура сайта

### Главная

- тёмный премиальный первый экран;
- заголовок "Презентации, которые производят впечатление";
- короткое описание услуг;
- карточки проектов;
- кнопка связи.

### Карточка проекта

На карточке:

- обложка;
- категория;
- 2-3 строки о задаче;
- кнопка "Смотреть кейс";
- кнопка "Открыть HTML";
- кнопка "Скачать PPTX" там, где это уместно.

### Страница кейса

- задача;
- что сделано;
- ключевые слайды;
- ссылка на HTML-демо;
- ссылка на PPTX/PDF;
- инструменты: Figma, PowerPoint, HTML;
- срок создания;
- призыв связаться.

## Кейсы на месяц

### Кейс 1. TechVision

Тип: коммерческое предложение для IT-компании.

Цель: главный флагманский кейс.

Слайды:

1. Обложка.
2. О компании.
3. Какие проблемы решаем.
4. Наше решение.
5. Продукты и услуги.
6. Почему выбирают нас.
7. Кейсы.
8. Этапы работы.
9. Коммерческое предложение.
10. Контакты.

### Кейс 2. Инвестиционный питч стартапа

Тип: презентация для инвесторов.

Показывает:

- структуру;
- цифры;
- рынок;
- визуализацию роста;
- уверенный деловой стиль.

### Кейс 3. Презентация IT-продукта

Тип: продуктовая презентация.

Показывает:

- интерфейсы;
- преимущества продукта;
- сценарии использования;
- современный технологичный стиль.

### Кейс 4. Годовой отчёт

Тип: отчёт компании с инфографикой.

Показывает:

- таблицы;
- графики;
- аккуратную работу с данными;
- корпоративный стиль.

### Кейс 5. Премиальная недвижимость

Тип: продающая презентация объекта.

Показывает:

- эмоциональный визуал;
- крупные изображения;
- премиальную подачу;
- коммерческий сторителлинг.

### Заготовки на будущее

- презентация медицинской клиники;
- образовательный проект;
- эффектная демо-презентация с анимацией.

## График на 4 недели

### Неделя 1

Результат недели: готова основа бренда, структура сайта и первый кейс.

Задачи:

- определить визуальный стиль портфолио;
- собрать структуру сайта;
- сделать дизайн кейса TechVision в Figma;
- подготовить PPTX для TechVision;
- сделать HTML-версию TechVision;
- экспортировать PDF и изображения слайдов.

### Неделя 2

Результат недели: на сайте уже 3 сильных проекта.

Задачи:

- закончить сайт до рабочей версии;
- сделать инвестиционный питч;
- сделать презентацию IT-продукта;
- для каждого кейса подготовить Figma, PPTX, HTML, PDF и изображения;
- добавить страницы кейсов на сайт.

### Неделя 3

Результат недели: портфолио выглядит убедительно и разнообразно.

Задачи:

- сделать годовой отчёт с инфографикой;
- сделать презентацию премиальной недвижимости;
- записать короткие видео-превью для всех готовых кейсов;
- улучшить главную страницу;
- проверить сайт на телефоне и компьютере.

### Неделя 4

Результат недели: проект опубликован и готов для Профи.ру.

Задачи:

- добавить 3 заготовки будущих кейсов как "скоро";
- вычитать все тексты;
- оптимизировать изображения и видео;
- опубликовать сайт;
- оформить профиль на Профи.ру;
- подготовить 3 варианта отклика на заказы;
- подготовить прайс на стартовые услуги.

## Профиль на Профи.ру

Что добавить:

- короткое позиционирование;
- 5-7 лучших изображений;
- ссылку на сайт-портфолио;
- описание услуги;
- понятные пакеты.

Пример пакетов:

- Базовая презентация: структура + дизайн 8-10 слайдов.
- Бизнес-презентация: 10-15 слайдов + PDF + PPTX.
- Премиум-кейс: Figma + PPTX + HTML + анимация.

## Очередность работы

1. Не начинать с сайта.
2. Сначала сделать TechVision как эталон качества.
3. Потом собрать сайт вокруг готового кейса.
4. Затем добавлять остальные кейсы.
5. Публиковать только когда есть минимум 3 полных кейса.

## Критерии готовности

Проект готов к запуску, когда есть:

- опубликованный сайт;
- минимум 3 полных кейса;
- Figma, PPTX и HTML для каждого полного кейса;
- изображения ключевых слайдов;
- короткие описания кейсов;
- рабочий профиль на Профи.ру;
- понятный прайс;
- шаблоны откликов.

## Первый шаг

Начать с кейса TechVision.

Сначала сделать не весь сайт, а один эталонный проект:

- сценарий 10 слайдов;
- тексты;
- визуальный стиль;
- Figma-макет;
- PPTX;
- HTML-версия.

После этого сайт будет строиться вокруг уже готового сильного материала.
